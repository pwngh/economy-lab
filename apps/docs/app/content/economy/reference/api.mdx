The library ships a curated, faceted [`exports`](https://nodejs.org/api/packages.html#exports) map: the main package carries the day-to-day surface, and each subpath is a focused entry point. Install it, then import from whichever one you need.

```bash
npm install @pwngh/economy-lab
```

```ts
import { createEconomy, topUp, toAmount } from '@pwngh/economy-lab';
import type { Store } from '@pwngh/economy-lab/ports';
```

## `@pwngh/economy-lab`

The main entry point. It carries the day-to-day surface — construction, operations, money, the ledger, outcomes and errors, and the principal types. The focused subpaths further down hold the port contracts, the adapter factories, the full worker and HTTP surfaces, and the testing harness.

### Construction

Stand up an `Economy` from a fully-built `Ports` bag — batteries-included in memory, or resolved from the environment (see [composition](#composition-and-environment)).

<SourceLink to="src/index.ts">src/index.ts</SourceLink>

| Export | What it is |
| --- | --- |
| `createEconomy` | The one call: assembles an Economy from a fully-built Ports bag, synchronously. |
| `memoryPorts` | A batteries-included in-memory Ports bag from one signing key; config, secrets, and store overridable at build. |
| `systemRuntime` | Bundles the four production runtime ports (clock, id generator, hasher, signer) from a signing key. |
| `Ports` | Every port the assembler needs, gathered into one readonly bag, secrets and config included. |
| `Runtime` | The four runtime ports systemRuntime returns: clock, ids, digest, signer. |
| `Economy` | The request-processing and read interface: submit operations, query state, check health, and close the store. |

<SourceLink to="src/money.ts">src/money.ts</SourceLink>

| --- | --- |

| `Currency` | The currencies the system handles: in-app CREDIT and real-world USD. |
| `toAmount` | Builds an Amount from currency and minor-unit count, throwing AMOUNT_OVERFLOW if outside 64-bit range. |

| `isAmount` | Type guard to check if a value is a branded Amount. |

| `negate` | Returns the negation of an amount, checking it stays in 64-bit range. |
| `compare` | Compares two amounts of the same currency, throwing CURRENCY_MISMATCH. |

| `isZero` | Checks if an amount is zero. |
| `isNegative` | Checks if an amount is negative. |
| `convertFloor` | Converts an amount to another currency at a rate, rounding down for safe outward payments. |
| `convertCeil` | Converts an amount to another currency at a rate, rounding up to under-cover risk like USD trust. |

## Focused entry points

Each subpath owns a slice of the surface. The principal **ports** types are also re-exported from the main package; everything else below lives only on its subpath — the **driver-carrying** adapters and the **engines** load a database or cloud driver, so they are imported only where that driver is installed.

| `Ports` | Every port `createEconomy(...)` needs, gathered into one readonly bag, secrets and config included. |
| `CallOptions` | Per-call options for the store and reads, such as an AbortSignal. |

| `StoredLink` | One posting as lineage returns it, carrying the two hashes that tie it into the account's tamper-evident chain. |
| `Movement` | One accepted in-instance movement: a balanced set of legs not yet posted to the ledger, made ledger-final at settle. |
| `EconomyEvent` | The fixed shape of every event the system emits, with type, version, subject, and audience. |
| `OutboxMessage` | One stored outbox row: an event plus the bookkeeping for delivering it. |
| `InboxMessage` | One stored inbox row: a verified inbound event mapped to the operation it applies. |
| `Sale` | A summary of a completed sale, recording the exact lines that posted so a refund can reverse them precisely. |
| `Saga` | The stored state of one in-flight payout, tracking reserve, rate, state, attempts, and settlement. |

| `Subscription` | The stored state of one recurring subscription: price, period, state, attempts, and the next due time. |
| `SubscriptionState` | One of the subscription lifecycle states, as a type. |

| `EntitlementGrant` | One non-revoked grant as EntitlementStore.list streams it: the sku and the expiry owns applies at read time. |

## `@pwngh/economy-lab/ops`

The in-process operations supervisor. It observes the composition only through the `meter` and
`logger` ports (wrapped by `createOpsRuntime`), detects twelve incident signatures with plain
threshold rules, and remediates through host-provided levers under mandatory guardrails —
cooldown, attempt cap into permanent escalation, and a containment latch on any integrity
episode. The core never imports this entry point; leaving the supervisor out of the composition
is the off switch. [Ops & runbooks](/economy/ops/) owns the operational story, with
[one runbook per signature](/economy/ops/runbooks/).

| Signature | Tier | Response |
| --- | --- | --- |
| `stuck-saga` | 1 | One guarded worker sweep, verified by re-reading the saga. |
| `outbox-backlog` | 1 | A targeted relay re-drive (`sweep({ only: ['relay'] })`), verified against the next gauge. |
| `inbox-dead-letter` | 1 | Revive the oldest dead rows; the normal drain re-applies them idempotently. |
| `integrity-mismatch` | 3 | Prove once, escalate, pause the worker, latch containment — never fix. |
| `deadlock-storm`, `engine-stall`, `treasury-breach`, `velocity-anomaly`, `signal-silence`, `retry-exhaustion`, `webhook-replay-storm`, `checkpoint-seal-slow` | 3 | Report or escalate with an advisory; no lever. |
